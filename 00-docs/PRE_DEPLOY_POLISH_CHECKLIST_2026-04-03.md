# Helius-002 Demo 部署前微调清单

> 用途：给 Coding AI 直接读取，在部署前做最后一轮轻量收尾  
> 目标：不再扩功能，不再大改结构，只做上线前必要的小修与统一

---

## 一、这轮目标

当前状态：
- 平台骨架已完成
- 第三轮 UI 已升级
- 第四轮功能小修已完成
- 已具备部署 demo-v1 的条件

这一轮不是继续做新功能，
而是：

**做部署前最后一轮轻量微调。**

---

## 二、明确要求

### 1. 删除 Dashboard 的 slogan 模块
用户最新反馈：
- slogan 板块当前效果不理想
- 容易破坏整体高级感
- 现阶段不是必须功能

#### 要求
- 删除 Dashboard 中与 slogan 相关的展示区
- 删除“换一句”“设为当前主页 slogan”等交互入口
- 删除与 slogan 相关的页面文案占位
- 保持 Dashboard 整体结构完整，不要因此留出奇怪空白

#### 技术要求
- 清理 Dashboard 中相关 DOM 渲染
- 清理事件绑定
- 清理与 slogan 相关的 localStorage 使用
- 清理未使用的状态与函数，避免遗留冗余代码

---

### 2. 做一轮中英混合文案统一
当前页面中存在一些中英混合表达，虽然有高级感，但略显不统一。

#### 建议策略
以中文为主，英文作为少量辅助标签，不要大面积中英混搭。

#### 优先统一这些区域
- Dashboard 标题区
- Script Delivery 相关文案
- Analysis Snapshot / Selected Title / Ready to use / Draft saved
- Library Overview / Ready Script 等标签

#### 要求
不要一刀切全改英文或全改中文，
而是让整体语言更统一、自然、适合中文场景 demo。

---

### 3. 再检查 5 个页面的视觉一致性
目标不是重做，而是统一。

#### 检查项
- 卡片圆角是否一致
- 按钮样式是否一致
- 状态标签样式是否一致
- 字体层级是否一致
- section title / section sub 的样式是否统一
- 各页留白节奏是否一致

#### 要求
如果发现个别页面明显跳风格，做轻量修正。

---

### 4. 收一轮 Dashboard
在删除 slogan 之后，Dashboard 仍需保持完整与高级感。

#### 检查以下模块是否还成立
- Hero 区
- 最近任务
- 工作流状态
- 素材库概览
- 最近产出预览

#### 要求
删除 slogan 后，不要让 Dashboard 变回空空的 landing page。
它仍然要像：

**成果驾驶舱 / 创作者控制台首页**

---

### 5. 检查 Import → Analysis → Script → Library 闭环
部署前必须再确认一次：

- Import 能进入 Analysis
- Analysis 能进入 Script
- Script 能保存到 Library
- Analysis 也可以单独保存到 Library
- Library 能正确展示 Analysis Only / Analysis + Script

如果有任何断头路，优先修掉。

---

### 6. 为部署做好最低准备
如果项目里还没有，需要补：
- 清晰的启动命令
- 本地构建命令
- 线上部署命令或说明
- README 中最简部署说明

目标不是做完整文档，而是让你和我后续都能快速验证线上版本。

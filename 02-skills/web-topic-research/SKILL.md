# Web Topic Research Skill

## 功能說明
圍繞一個視頻選題或創作者自己的感悟，進行網上資料收集，並整理成可供後續創作使用的研究包。

這個 Skill 不直接寫完整腳本。它的產物是選題研究、素材庫、來源清單和視頻方向建議，供 `script-generator`、`reference-script-rewriter`、`long-video-storyboard` 繼續使用。

## 適用場景
- 用戶有一個初步觀點，想查網上是否有支撐材料。
- 用戶想知道其他創作者如何表達類似觀點。
- 用戶想為下一個視頻方向建立素材庫。
- 用戶需要把研究結果保存到 `04-outputs/` 的單獨文件夾。
- 用戶需要反面觀點，避免視頻過度單邊。

## 核心原則
- 先保留用戶自己的原始洞察，不要被外部資料稀釋。
- 搜集資料時分清：創作者表達、研究支撐、案例素材、反面觀點。
- 不把網文當成權威研究。
- 不使用來源不可靠的精確數字。
- 對不確定的事標記「需要核實」。
- 最終輸出要能直接服務視頻創作，而不是堆鏈接。

## 輸入
| 參數 | 類型 | 必填 | 說明 |
|------|------|------|------|
| topic | string | 是 | 研究主題或視頻方向 |
| user_insight | string | 否 | 用戶自己的原始感悟、模型、草稿 |
| target_audience | string | 否 | 目標受眾，默認按中文知識類長視頻處理 |
| output_dir | string | 否 | 默認 `04-outputs/{slug}-research/` |
| depth | string | 否 | `light` / `standard` / `deep`，默認 `standard` |
| language_scope | string | 否 | `zh` / `en` / `both`，默認 `both` |

## 輸出
默認生成一個獨立文件夾：

1. `01-research-notes.md`
   - 用戶原始洞察
   - 網上素材整理
   - 可借鑑觀點
   - 研究支撐
   - 反面觀點
   - 比喻、案例、金句候選

2. `02-video-direction.md`
   - 暫定標題
   - 核心主張
   - 視頻結構
   - 主線建議
   - 視覺設計方向
   - 可用 Hook

3. `03-source-list.md`
   - 來源鏈接
   - 每個來源的用途
   - 可引用程度
   - 後續搜索關鍵詞

## 工作流程

### Step 1：提煉用戶原始洞察
先把用戶自己的觀點整理成清楚表述。

必須輸出：
- `原始主張`
- `核心模型`
- `情緒張力`
- `最值得保留的個人洞察`
- `需要外部資料支撐的部分`

### Step 2：制定搜索方向
至少分成 4 類搜索：

1. 創作者表達
   - 其他創作者如何講類似觀點
   - 相關框架、標題、比喻

2. 研究或理論支撐
   - 學術研究
   - 經典理論
   - 書籍或心理學/教育學材料

3. 反面觀點
   - 這個觀點有什麼限制
   - 哪些情況下不成立
   - 如何避免講得太絕對

4. 案例和比喻
   - 生活案例
   - 創作案例
   - 可視化比喻

### Step 3：網上搜索
需要上網搜索時，優先找：
- 原始研究或可信研究機構
- 作者本人文章或視頻頁
- 大學、期刊、專業機構
- 可確認來源的書籍或訪談

避免：
- 無來源的二手數字
- 「某研究表明」但不給研究名的文章
- 沒有上下文的名人語錄
- 無法驗證的成功學故事

### Step 4：整理成可創作素材
不要只貼鏈接。每個素材都要回答：
- 它能支撐視頻哪一段？
- 它可以轉成什麼說法？
- 它有什麼風險？
- 是否可以做成畫面或比喻？

### Step 5：形成視頻方向
把研究結果壓縮成一條可拍的主線。

必須包含：
- 1 個推薦主標題
- 3-5 個備選標題
- 1 句核心主張
- 1 條推薦主線
- 6-8 分鐘視頻結構
- 可視化方向
- 反面觀點或限制條件

### Step 6：保存到 output
所有文件寫入獨立文件夾，不覆蓋舊研究。

文件夾命名建議：
`04-outputs/{topic-slug}-research/`

## 質量檢查
輸出前自檢：
- 是否保留並強化了用戶原始洞察？
- 是否區分了創作者觀點和研究支撐？
- 是否有反面觀點，避免單邊論證？
- 是否每個來源都說明了用途？
- 是否避免使用未核實的精確數字？
- 是否形成了可直接進入腳本創作的視頻主線？
- 是否保存到獨立 output 文件夾？

## 與其他 Skill 協作
- **上游**: 用戶原始感悟、選題草稿、`materials-collector`。
- **下游**: `script-generator`，用研究結果生成完整腳本。
- **下游**: `reference-script-rewriter`，當研究包含參考腳本時可繼續重寫。
- **下游**: `long-video-storyboard`，腳本完成後生成分鏡。
- **下游**: `packaging-optimizer`，用研究中的標題和 Hook 做發布包裝。

## 使用示例

### 示例 1：根據個人感悟做選題研究
> 我想做一支「為什麼輸出大於輸入」的視頻。這是我的模型：改變量 > 行動量 > 思考量 > 學習量。請上網收集資料，整理成 output 的單獨文件夾。

輸出：
- `04-outputs/輸出大於輸入_選題研究包/01-research-notes.md`
- `04-outputs/輸出大於輸入_選題研究包/02-video-direction.md`
- `04-outputs/輸出大於輸入_選題研究包/03-source-list.md`

### 示例 2：先研究，不寫腳本
> 幫我查一下「注意力管理比時間管理更重要」這個方向，看看有哪些研究、創作者觀點和反面觀點，先不要寫腳本。

執行：
- 只生成研究包
- 不進入完整腳本創作

## 默認假設
- 用戶要的是下一支視頻的方向材料，不是百科報告。
- 研究結果應該服務創作判斷，包括標題、主線、比喻、結構和風險提示。
- 如果資料不足，直接標明不足，不硬湊來源。

## 二、本轮只做 3 个功能

# 功能 1：Analysis 页面增加“保存报告”功能

## 需求背景
目前 Analysis 只作为中间过程存在，
但实际上分析报告本身就是内容资产。

如果不能保存，就会导致：
- 分析结果无法沉淀
- Library 只存脚本，不存分析
- 资产库价值不完整

## 目标
让 Analysis 结果可以单独保存到 Library。

## 具体要求

### 在 Analysis 页面增加两个按钮
- `保存分析报告`
- `保存并继续生成脚本`

### 逻辑说明
#### 按钮 1：保存分析报告
点击后：
- 将当前 analysis 结果保存到 Library
- 即使没有 script，也允许保存
- Library 中显示为一个分析类资产

#### 按钮 2：保存并继续生成脚本
点击后：
- 先保存当前 analysis
- 然后进入 Script 生成流程

## Library 中的展示要求
Library 需要支持两种资产类型：
- `Analysis Only`
- `Analysis + Script`

每条资产应明确显示：
- 类型标签
- 是否已生成脚本
- Analysis snapshot
- 若有脚本，则显示 Script snapshot

## 注意
- 不要因为加入这个功能，把 Analysis 页变复杂
- 按钮位置自然，仍然保持页面干净

# 功能 2：Script 页面增加“输出风格选择”

## 需求背景
现在 Script 只有一种默认输出风格。
但对于创作者平台来说，风格切换是一个非常有价值的产品能力。

## 当前这一轮的策略
### 先做“手动选择风格”版本
不要现在直接做复杂的 style skill 自动生成。

## 具体要求

### 在 Script 页面增加一个风格选择模块
形式可以是：
- tab
- segmented control
- 下拉选择器

### 先内置 3~5 种风格
建议预设：
- 理性拆解风
- 朋友聊天风
- 冷静专业风
- 高能短视频风
- MoneyXYZ 风

### 行为逻辑
- 用户在 Script 页面选择风格
- 重新生成对应风格的脚本（当前阶段可以用 mock / 模板映射）
- 标题与脚本内容根据风格变化

### 输出要求
风格变化至少体现在：
- 标题语气变化
- Hook 变化
- CTA 变化
- 语气变化

## 注意
- 这一轮不要做复杂的 AI 风格学习系统
- 不要生成真实 skill 文件
- 只做“可选风格输出”这件事

## 可扩展设计（先预埋，不全做）
可以在 UI 上留一个轻量入口，例如：
- `从参考文章提取风格（Coming Soon）`

但不要这轮完整实现。

# 功能 3：Dashboard 增加 slogan 扭蛋切换功能

## 需求背景
平台首页目前功能性够了，但还缺一点“人格感”和“创作动机感”。

这个功能的目标是：
- 让首页更有生命力
- 让创作者打开平台时有情绪价值
- 增加产品记忆点

## 功能目标
在 Dashboard 首页增加一个 slogan 随机切换功能。

## 交互要求

### 基本功能
- 显示一句 slogan
- 有一个按钮：`换一句`
- 点击后随机切换一句新 slogan

### 进阶功能
- 可加一个按钮：`设为当前主页 slogan`
- 用户选中喜欢的一句后可以固定
- 固定内容保存到 localStorage

## 呈现风格
- 不要做得花哨
- 不要像营销文案轮播
- 要像“创作者今天开工前的状态提示”

## slogan 内容建议
分 3 类：

### A. 创作激励型
- 不是你没灵感，是你没有工作流。
- 先把一条内容做通，比做十条半成品更重要。
- 内容不是堆出来的，是拆出来的。

### B. 认知反差型
- 大多数人不是不会创作，而是不会沉淀资产。
- 你缺的不是努力，而是结构。
- 不是每一条内容都要爆，但每一条都该留下资产。

### C. 冷静专业型
- 把输入转化成结构，把结构转化成产出。
- 工作流越清晰，创作焦虑越少。
- 今天的任务，不是发内容，而是建立系统。

## 注意
- slogan 切换功能应该轻量、优雅
- 不要影响 Hero 区主结构
- 不要喧宾夺主

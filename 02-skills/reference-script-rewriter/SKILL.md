# Reference Script Rewriter Skill

## 功能说明
把一篇已有视频脚本转化为一篇新的原创视频脚本。重点不是同义改写，而是先拆出原视频的核心洞察、结构方法和可借鉴技巧，再重新寻找一个新的主线、标题和表达框架。

这个 Skill 适合做：
- 从爆款视频脚本中学习选题和结构
- 根据参考视频生成同主题但不同角度的新长视频脚本
- 把原视频要点变成自己的口播稿
- 生成可继续交给 `long-video-storyboard` 的新脚本

## 核心原则
- 借鉴洞察，不照搬表达。
- 保留主题相关性，但必须换主线、换例子、换叙述顺序。
- 先提炼，再重写，不直接进入改写。
- 不编造精确事实、数据、来源或个人经历。
- 新脚本要能独立成立，不能像原视频的摘要。
- 默认输出中文长视频口播，预计 5-10 分钟。

## 输入
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reference_script | string | 是 | 原始参考视频脚本或脚本文件路径 |
| target_topic | string | 否 | 用户指定的新主题；没有则从参考脚本中提炼 |
| target_audience | string | 否 | 默认 `中文知识类观众 / 创作者 / 自由职业者 / 副业人群`，按内容调整 |
| output_dir | string | 否 | 默认写入 `04-outputs/{slug}/` |
| style_bias | string | 否 | 默认 `中文长视频深度解说 / 口语化 / 低说教感` |
| generate_storyboard | bool | 否 | 默认 `true`，是否继续生成分镜 |
| humanize | bool | 否 | 默认 `true`，是否额外输出更自然的人性化版本 |

## 输出
默认输出一个独立文件夹，包含：

1. `01-key-points.md`
   - 原视频核心主题
   - 可借鉴洞察
   - 不应照搬的表达
   - 新脚本主张
   - 新标题候选

2. `02-rewritten-video-script.md`
   - 完整新视频脚本
   - 标题、定位、受众、核心观点
   - 5-10 分钟口播正文

3. `02-rewritten-video-script-humanized.md`
   - 可选输出
   - 更像真人口播的版本
   - 不覆盖原脚本

4. `03-storyboard.md`
   - 可选输出
   - 基于新脚本生成的导演分镜、B-roll、字幕和剪辑提示

## 工作流程

### Step 1：读取参考脚本
先确认脚本来源和文本完整性。

如果用户给的是文件路径：
- 读取原文件
- 不修改原文件
- 保留来源路径写入 `01-key-points.md`

如果脚本过长：
- 先按标题、段落、时间线或语义自然分块
- 不按字数硬切

### Step 2：提炼原视频
先输出原视频要点，不急着写新稿。

必须提炼：
- `核心主题`：原视频真正讨论的问题
- `核心破局点`：它反常识或有记忆点的地方
- `结构骨架`：开场、论证、方法、案例、收尾如何推进
- `可借鉴技巧`：口号、比喻、脆弱性展示、案例、转场、变现连接等
- `可复用洞察`：可以借走的思想
- `不可照搬内容`：原作者个人经历、独特表达、具体口头禅、专属社群转化话术

### Step 3：寻找新主线
不要直接把原脚本换词。先给新脚本找一个可以独立成立的主张。

新主线必须满足：
- 与原主题相关
- 角度不同
- 受众更明确或更普适
- 能支持一篇完整长视频
- 一句话可以说清楚

常见转换方式：
- 从个人经验转为普适系统
- 从方法清单转为认知框架
- 从一个职业场景转为更广泛人群
- 从「问题批判」转为「解决方案」
- 从「如何做」转为「为什么你一直做错」

### Step 4：重写新脚本
按新主线重新写，不照原顺序搬运。

默认结构：
1. Hook：提出反直觉判断或强共鸣场景
2. 问题：解释观众为什么会卡住
3. 核心观点：给出新的主张
4. 框架：拆成 3-5 个部分，每个部分有场景或例子
5. 情感收束：承认现实复杂性，避免像训话
6. 行动建议：给观众一个可以当天执行的微行动

写作要求：
- 口语化，但不要水。
- 每 60-90 秒至少有一个具体场景。
- 抽象概念必须落到生活画面或比喻。
- 不连续堆 3 个以上金句。
- 例子可以是典型情境，但不能伪装成真实新闻或真实个人经历。
- 如果使用公共事实、书籍或人物案例，不确定时要标注需要核实。

### Step 5：人性化版本
当 `humanize=true` 时，单独输出人性化版本。

处理重点：
- 减少过度整齐的结构感
- 增加自然停顿、转折、自我修正
- 删除过度像标语的句子
- 保留核心观点和段落功能
- 不覆盖 `02-rewritten-video-script.md`

### Step 6：生成分镜
当 `generate_storyboard=true` 时，把新脚本交给 `long-video-storyboard` 的逻辑处理。

分镜必须包含：
- 内容骨架诊断
- 6 段视频结构
- 导演分镜表
- B-roll 素材清单
- 素材可得性
- 低成本替代方案
- 剪映执行提示

## 质量检查
输出前自检：
- 是否先提炼原视频，而不是直接改写？
- 新主线是否能一句话说清楚？
- 新脚本是否换了标题、例子、顺序和表达？
- 是否避免复制原视频独特口头禅、个人经历和专属转化话术？
- 是否至少提供 3 个新标题候选？
- 是否包含明确受众和核心观点？
- 是否有具体场景，而不只是抽象说理？
- 如果生成分镜，是否包含素材可得性和低成本替代方案？
- 是否所有输出都放在独立文件夹，且不破坏原文件？

## 与其他 Skill 协作
- **上游**: `transcription` / `video-download`，用于获取视频脚本文本。
- **下游**: `humanizer-zh`，用于生成更自然的人性化口播版本。
- **下游**: `long-video-storyboard`，用于把新脚本拆成分镜和剪映执行方案。
- **下游**: `content-repurposer`，用于把完成的新脚本继续拆成短视频、小红书、推文等平台内容。

## 使用示例

### 示例 1：根据参考脚本生成新脚本和分镜
> 根据 `03-inputs/某视频/script.md` 的要点，重新写一篇相关的视频脚本，然后生成视频分镜，放在 output 的单独文件夹。

输出：
- `04-outputs/{slug}/01-key-points.md`
- `04-outputs/{slug}/02-rewritten-video-script.md`
- `04-outputs/{slug}/02-rewritten-video-script-humanized.md`
- `04-outputs/{slug}/03-storyboard.md`

### 示例 2：只做新脚本
> 参考这篇脚本，帮我重新找一个新角度写一篇 8 分钟口播稿，先不要分镜。

执行：
- `generate_storyboard=false`
- 只输出要点提炼和新脚本

## 默认假设
- 用户要的是「可借鉴、可复用、可继续生产」的脚本资产，不是摘要。
- 如果用户没有指定新角度，优先从原视频中提炼一个更普适的新主张。
- 如果用户要求「human」，只新增人性化版本，不覆盖原文件。
- 如果用户说「再生成分镜」，默认调用 `long-video-storyboard` 的标准输出结构。

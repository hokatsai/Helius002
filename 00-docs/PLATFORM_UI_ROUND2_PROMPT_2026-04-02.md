## 三、第二轮重点优化模块

## 1. Dashboard → 升级为“成果驾驶舱”

### 当前问题
Dashboard 现在更像 landing page / 欢迎页，
还不像一个创作者每天会打开的工作台首页。

### 优化目标
把 Dashboard 做成：
**有任务感、有状态感、有成果感的驾驶舱首页。**

### 需要新增/优化的内容

#### 模块 A：Hero 区保留，但更有控制台气质
保留：
- 标题
- 一句话定位
- CTA

但优化：
- 字体层级更明确
- 间距更高级
- CTA 更突出
- Hero 区不要只像宣传页，要像“工作台入口”

#### 模块 B：加入「最近任务」卡片
展示：
- 最近一次任务名称
- 来源类型（YouTube / 文本）
- 当前状态（分析完成 / 脚本已生成 / 已保存）
- 快速跳转按钮（继续查看）

#### 模块 C：加入「工作流状态」卡片
展示：
- Import → Analysis → Script → Library
- 当前卡在哪一步
- 用简洁的步骤条或阶段卡表示

#### 模块 D：加入「素材库概览」卡片
展示：
- 已保存任务数量
- 最近保存时间
- 最近脚本是否 ready

#### 模块 E：加入「最近产出预览」
展示：
- 最近一次脚本标题
- 一小段脚本 preview
- 让观众一眼看到“这平台真的产出了东西”

### Dashboard 最终效果要求
首页看起来必须像：
- Linear / Notion / 现代 SaaS 后台首页
- 有状态，有任务，有结果
- 不只是三张介绍卡

---

## 2. Import 页 → 从“表单页”升级为“任务启动页”

### 当前问题
Import 页功能是对的，但现在更像普通输入表单。

### 优化目标
让它看起来像：
**一次任务的正式开始界面。**

### 需要优化
- 输入区更清晰
- URL 输入区与文本输入区视觉分层
- 右侧状态区更完整

### 右侧状态区建议增强为：
- 当前任务名称
- 来源类型
- 创建时间
- 当前阶段
- 下一步提示
- 一个简易的流程进度条

例如：
- Step 1 Import ✅
- Step 2 Analysis ⏳
- Step 3 Script
- Step 4 Library

这样观众会明显感觉：
**不是在填表，而是在启动一个 AI 工作流。**

---

## 3. Analysis 页 → 提升为“拆解工作台”

### 当前问题
Analysis 的内容已经有了，但还是偏信息堆叠。

### 优化目标
把它做成：
**一个真正的内容拆解台。**

### 结构优化建议

#### 上半区：两栏布局
左边：
- Summary
- Why it works

右边：
- Key insights

#### 下半区：Viral Structure 单独做成步骤卡
每一项都做成明确模块：
- Hook
- Evidence
- Argument
- Twist + CTA

卡片之间层级明确，不要只是同质区块。

### 按钮区优化
底部 CTA “生成脚本” 要更明显：
- 主色按钮
- 更有下一步推进感

### 要达到的感觉
这一页一出来，观众会觉得：
> 哦，这个平台真的在帮我拆内容，而不是只是展示一段文案。

---

## 4. Script 页 → 提升为“可直接使用的结果页”

### 当前问题
逻辑是对的，但结果页的“可用感”和“成果感”还不够强。

### 优化目标
让 Script 页看起来像：
**一个已经能交付给创作者使用的结果页面。**

### 需要优化的地方

#### 标题备选区
- 做成更明显的候选卡片
- 每个标题像可以被直接选中的结果项
- 不要只是普通文字列表

#### 脚本正文区
- 结构标签（Hook / Evidence / Core / Twist / CTA）更醒目
- 每个模块之间留白更大
- 更像专业编辑页面

#### 成果状态增强
可加一个小标签：
- Ready to use
- Script generated
- Draft saved

#### 操作区优化
按钮：
- 复制脚本
- 保存到素材库

这两个按钮要更像“最后一步交付动作”，
而不是普通按钮。

---

## 5. Library 页 → 提升为“资产沉淀页”

### 当前问题
Library 已经能看，但还像任务记录，不像资产库。

### 优化目标
让这一页更像：
**创作者自己的内容资产沉淀空间。**

### 需要优化
- 每张任务卡更像一个资产条目
- 卡片信息分层更清楚
- 增加一些更专业的标签感

### 每张卡片建议突出：
- 任务名称
- 来源类型
- 创建时间
- Analysis snapshot
- Script snapshot
- 当前状态（Saved / Ready）

### 观众感知目标
观众看到这页会觉得：
> 这个平台不是一次性生成器，而是会越用越有价值的内容系统。

---

## 四、统一视觉要求

### 风格关键词
- 白色 / 浅灰
- 极简
- 高级感
- 现代 SaaS
- Notion / Linear / Arc 风格

### 视觉细节要求
- 更强的留白
- 更清晰的字体层级
- 更统一的卡片系统
- 更克制的阴影
- 更稳定的按钮风格
- 页面节奏更舒服

### 不要做成这样
- 不要花哨炫技
- 不要重渐变
- 不要赛博风
- 不要开发者控制台感过重
- 不要把每个区块都做得一模一样

---

完成后，平台应该达到：

### 1. Dashboard 像控制台，而不是欢迎页
### 2. Analysis 像拆解工作台

### 4. Library 像资产沉淀区
### 5. 整体更适合上台展示

观众应该能感觉到：

**“这已经不像原型了，而像一个真正的创作者平台。”**

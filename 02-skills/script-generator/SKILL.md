# Script Generator Skill（Multi-Style Engine）

## 功能说明
本 Skill 是 Helius-002 系统的核心脚本引擎。支持多创作者风格切换，可根据不同风格模板生成高完成度脚本。

> ⚠️ **核心系统准则 (User Preference Caching)**:
> 根据使用者设定的永久偏好，**未来所有由于 /money 或默认写脚本命令触发的文案，统一以"深度长视频解说文案（预计 5-10 分钟时长，文本量丰富详实，包含深度论证与丰满的情感起伏）"为唯一默认输出标准**。不再输出一两分钟的短口播。

## 支持的创作者风格

### 🔺 三角结构风格（默认）
**核心哲学：三角结构**
1. **认知反差 (Contrarian Truth)**：从大众认知的反面切入，打破思维惯性
2. **理性证据 (Rational Evidence)**：引用权威数据、经典书籍或科学实验
3. **感性共鸣 (Emotional Resonance)**：朋友般的坦诚交流，提供情感锚点
4. **现实锚点 (Concrete Cases)**：优先使用真实案例、个人经历、生活场景，避免整篇停留在抽象大道理

**五步脚本结构**：
1. **黄金开场 Hook**：反直觉结论 + 视频路标指引（前 3 秒留人）
2. **权衡论证 Evidence**：引入权威背书或数据，并落到现实案例
3. **核心逻辑 Core Argument**：模块化拆解 + "降维类比" + 微型故事/场景
4. **情感反转 Twist**：个人经验的"阴影面"坦白，或一个能让观众代入的具体事件
5. **行动呼吁 CTA**：具体的"微行动"建议

**风格源**：`02-skills/triangle-narrative/SKILL.md`

### 🧠 Dan Koe 风格（新增 ⬆️）
**核心哲学：P3 框架 + 高信息密度**
1. **Purpose（目的）**：创造一个会进化的目的，不是找"命中注定的使命"
2. **Path（路径）**：技能栈组合 + 自我教育 + 每日实践
3. **Priority（优先级）**：量化任务作为反馈回路

**脚本结构**：
1. **开场**：哲学级洞察 + 个人立场声明
2. **框架引入**：命名一个"框架"（如 P3、HUMAN 3.0）
3. **层层递进**：从表层 → 深层 → 底层逻辑
4. **自我变现连接**：如何把认知变成收入
5. **收尾**：回到开场观点的升维版本

**风格源**：`02-skills/dankoe/SKILL.md`

### 🏪 認知便利店M 风格（新增 ⬆️）
**核心特征**：中文长视频深度解说、认知升级、B站风格

**脚本结构**：
1. **引入**：日常场景引发的认知冲突
2. **展开**：多信源交叉论证（书籍 + 论文 + 个人经历）
3. **深度段**：一个能"改变观众世界观"的核心论点
4. **收束**：回到现实生活的可操作建议

**风格源**：`02-skills/renzhi-convenience-store/`

## 斜杠指令系统

### 原有指令
| 指令 | 风格 | 功能 |
|------|------|------|
| `/money [主题]` | 三角结构 | 全量生成五步结构脚本 |
| `/hook [主题]` | 通用 | 生成 5 个认知反差爆款标题 |
| `/refine [文本]` | 通用 | 优化现有文案，植入情感锚点 |
| `/humanize [文本]` | 通用 | (新增) 去除 AI 味，增加人性化色彩 |
| `/check [脚本]` | 通用 | 检查脚本质量（引用/逻辑/说教感） |

### 新增指令（⬆️）
| 指令 | 风格 | 功能 |
|------|------|------|
| `/dankoe [主题]` | Dan Koe | 生成 P3 框架 + 高信息密度脚本 |
| `/renzhi [主题]` | 認知便利店M | 生成中文长视频深度解说脚本 |
| `/hooks [主题]` | 全风格 | 同一主题生成 5+ 种不同风格开场变体 |
| `/short [主题]` | 自动选择 | 生成 30s/60s/90s 短视频脚本 |

## 核心能力

### 原有能力
- 三角结构风格脚本生成
- 爆款标题生成（/hook）
- 现有文案优化（/refine）
- 脚本自检（/check）
- 多平台适配（TikTok/Bilibili/YouTube）

### 新增能力（⬆️）
- **多风格切换**：通过 `--style` 参数或斜杠指令选择创作者风格
- **创作者模板热加载**：运行时从 `02-skills/triangle-narrative/`、`dankoe/`、`renzhi-*/` 读取风格定义
- **短视频脚本协议**：标准化短视频模板（30s / 60s / 90s）
- **Hook 变体批量生成**：同一主题 5+ 种不同风格开场
- **段落级结构编辑**：`/refine` 支持只替换指定段落而非整篇重写
- **AI 痕迹擦除 (Humanizer)**：生成后可选 `/humanize` 指令进行去 AI 味处理
- **长短双版本输出**：一次生成长视频 + 短视频两个版本

## 输入/输出规范

### 输入
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| topic | string | 是 | 视频主题/核心观点 |
| audience | string | 否 | 目标受众 |
| user_cases | string | 否 | 用户提供的真实案例/经历/人物故事，生成时优先使用 |
| slash_command | string | 否 | /money /dankoe /renzhi /hook /hooks /refine /check /short |
| style | string | 否 | `triangle`（默认）/ `dankoe` / `renzhi` / `auto` |
| duration | string | 否 | `long`（默认，5-10分钟）/ `medium`（3-5分钟）/ `short`（30-90秒） |
| reference_text | string | 否 | 参考文案（用于 /refine） |
| target_section | string | 否 | 指定段落名（用于段落级 /refine） |

### 输出
| 参数 | 类型 | 说明 |
|------|------|------|
| titles | list | 3-5 个爆款标题选项 |
| script | string | 完整脚本（含画面提示、口播词，且至少含 2 个具体案例/场景） |
| script_short | string | 短视频版本（仅当 duration=long 时附带） |
| check_report | object | 自检报告（认知反差值/逻辑闭环度） |
| style_used | string | 实际使用的风格标识 |

## 案例注入规则（新增）

### 优先级
1. **用户真实案例优先**
   - 如果用户明确给了自己的经历、身边人的故事、项目经历、失败教训，优先使用这些素材。
   - 改写时保留事件骨架，只优化叙述张力，不篡改关键事实。

2. **公共真实案例次之**
   - 如果没有用户案例，可使用常见公开案例、经典实验、作者/企业/创作者案例。
   - 避免编造精确数字或伪造出处。

3. **典型生活场景兜底**
   - 如果既没有用户案例，也不适合引入公共案例，就补一个高代入感的生活化小故事。
   - 这类内容应写成“典型情境”或“你一定见过这种人”，不要伪装成真实新闻。

### 最低案例要求
- 长视频脚本：至少 2 个具体案例
- 短视频脚本：至少 1 个具体案例或场景
- 每个核心观点尽量配 1 个“人/场景/事件”，避免连续两段都只有抽象论述

### 案例类型白名单
- 个人经历：开会卡壳、拖延复发、第一次做作品被否定、复盘后修正
- 身边人物：朋友报课不行动、同事会做笔记但讲不清楚、学生刷题不复盘
- 公开人物：创作者、创业者、投资人、运动员、经典作者
- 生活场景：地铁刷收藏夹、买课后不看、会议临场表达、深夜做笔记但第二天不用
- 经典研究：棉花糖实验、生成效应、费曼技巧、习惯形成相关研究

### 写作约束
- 不允许整篇只有观点、定义、金句，没有人物或场景
- 不允许为了“真实感”编造过细节的假事实
- 如果案例是合成的，写法应偏“典型情境”，而不是“我有个朋友去年 3 月在某公司...”
- 案例的作用是承接观点，不是喧宾夺主；一个案例讲透一个点即可

## 短视频脚本协议（新增 ⬆️）

### 30 秒模板
```
[Hook] 1 句（3秒）
[核心观点] 1-2 句（10秒）
[证据/例子] 1 句（10秒）
[CTA/金句收尾] 1 句（7秒）
```

### 60 秒模板
```
[Hook] 1-2 句（5秒）
[问题/痛点] 1-2 句（10秒）
[核心观点] 2-3 句（15秒）
[证据/故事] 2-3 句（15秒）
[反转/情感点] 1 句（8秒）
[CTA] 1 句（7秒）
```

### 90 秒模板
```
[Hook] 1-2 句（5秒）
[问题/痛点] 2 句（10秒）
[观点 1] 2-3 句（15秒）
[观点 2] 2-3 句（15秒）
[故事/案例] 3-4 句（20秒）
[情感反转] 1-2 句（10秒）
[CTA + 金句] 1-2 句（15秒）
```

## 风格自动选择逻辑（新增 ⬆️）

当 `style=auto` 时，根据主题关键词自动匹配：

```
if 主题涉及 "金钱/理财/投资/财富/反脆弱/认知偏差":
    → 三角结构风格
elif 主题涉及 "创作者/一人公司/个人品牌/内容创作/副业":
    → Dan Koe 风格
elif 主题涉及 "认知升级/哲学/心理学/社会观察" 且目标平台=B站:
    → 認知便利店M 风格
else:
    → 三角结构风格（默认）
```

## 执行流程
1. 解析斜杠指令或直接传入参数
2. 确定风格（指定 / auto / 默认三角结构）
3. 收集案例素材（用户真实案例 → 公共真实案例 → 典型生活场景）
4. 从对应创作者 Skill 目录加载结构模板
5. 构建 prompt（含风格模板 + 用户主题 + 受众 + 案例素材）
6. 调用 LLM 生成内容
7. 如果 duration=long，同时生成短视频版本
8. 执行 /check 自检，重点检查“案例密度”与“假大空风险”
9. 输出最终脚本到 `04-outputs/scripts/`

## 错误处理
### 错误类型与降级方案
| 错误类型 | 降级方案 | 重试机制 |
|----------|----------|----------|
| LLM API 超时 | 返回基础模板脚本，标记需人工审核 | 3 次重试 |
| LLM API 配额用尽 | 回退到预设模板库选择 | 不重试 |
| 生成内容为空 | 返回错误提示 + 建议修改方向 | 3 次重试 |
| 风格模板文件缺失 | 回退到默认三角结构风格 | 不重试 |

### 错误日志
所有错误自动记录到 `06-logs/script-generator/YYYY-MM-DD.log`

## 使用示例

### 示例1：/money 完整脚本（MoneyXYZ 默认）
```bash
helius.py script "为什么年轻人不应该追求稳定的工作" --command /money
```

### 示例2：/dankoe 风格脚本
```bash
helius.py script "如何用2小时建立内容生态" --command /dankoe
```

### 示例3：/renzhi 风格脚本
```bash
helius.py script "认知偏差如何控制你的消费行为" --command /renzhi
```

### 示例4：/hooks 多风格开场变体
> 输入: /hooks 延迟满足
> 输出:
> - [三角结构] "延迟满足是 20 世纪最大的谎言"
> - [Dan Koe] "你不需要延迟满足，你需要的是重新定义'满足'"
> - [認知便利店] "斯坦福棉花糖实验被推翻了，但没人告诉你真相"

### 示例5：/short 短视频脚本
```bash
helius.py script "复利思维" --command /short --duration 60
```

### 示例6：/hook 爆款标题
> 输入: /hook 理财
> 输出:
> - "理财的第一步，90% 的人都做错了"
> - "我不是教你抠门，是教你用别人的钱赚钱"

# Long Video Storyboard Skill

## 功能说明
把文案拆解为适用于 6-10 分钟长视频的分镜脚本，重点不是“持续刺激”，而是通过节奏波形、视觉锚点和认知负担控制，让观众持续沉浸、能理解、也有时间消化。

这个 Skill 默认面向中文解说型长视频，适配 YouTube / Bilibili / 知识类个人频道。
它有两个输出层级：
- `导演分镜模式`：偏结构、节奏、镜头设计
- `剪映执行模式`：偏 CapCut 新手逐步操作清单，并拆成“粗剪清单 + 精剪补充方案 + 最终检查清单”

## 什么时候用
- 用户已经有完整文案，想拆成可执行分镜
- 用户只有大纲，但需要导演级节奏设计
- 用户强调“沉浸感”“别太吵”“降低理解成本”“给剪辑师可直接执行的表”
- 用户需要把抽象概念变成图像、比喻、案例、场景
- 用户希望“不要讲原理，只告诉我在剪映里点哪里”
- 用户是不会剪辑的新手，需要照着步骤完成

## 核心目标
- 保持观众沉浸，而不是连续刺激
- 节奏必须有波形：高 → 中 → 低 → 再拉高
- 降低理解成本，但保留必要的思考空间
- 每个关键观点必须有视觉锚点
- 每 30-60 秒至少设计一次注意力回拉点
- 避免连续高强度剪辑超过 60 秒

## 输入
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| script | string | 是 | 原始文案或逐段文稿 |
| duration_target | string | 否 | 默认 `6-10min` |
| platform | string | 否 | 默认 `YouTube` |
| style_bias | string | 否 | 默认 `深度解说 / 沉浸叙事` |
| output_language | string | 否 | 默认 `zh-CN` |
| mode | string | 否 | `storyboard`（默认）/ `capcut-execution` / `both` |
| material_budget | string | 否 | 默认 `low`，可选 `low` / `medium` / `high`，用于决定 B-roll 和视觉素材方案的复杂度 |
| editor_level | string | 否 | 默认 `beginner`，可选 `beginner` / `intermediate`，用于决定剪映操作说明细度 |

## 输出
当 `mode=storyboard` 时，输出导演分镜。
当 `mode=capcut-execution` 时，输出三份剪映文件：
- `粗剪清单`
- `精剪补充方案`
- `最终检查清单`

当 `mode=both` 时，先输出导演分镜，再输出这三份剪映文件。

### A. 导演分镜输出

#### 0. 内容骨架诊断
在正式分镜前，必须先输出一份简短诊断，避免机械按时间切段。

诊断必须包含：
- `主问题`：这条视频到底在回答什么问题
- `核心观点`：2-4 个最关键的推进点
- `观众理解门槛`：哪些概念、背景或转折可能让观众掉线
- `信息密度风险`：哪些段落可能太满，需要降速或拆开
- `案例/场景缺口`：哪些观点需要补一个人、一个场景或一个具体例子
- `视觉化机会`：哪些抽象概念最适合做图示、隐喻或 B-roll
- `可剪辑金句`：3-5 句适合做关键词字幕、截图或重点放大的句子

#### 1. 视频结构
固定用以下 6 段结构，如果素材天然更长，可细分为 7-8 段，但仍需保持这条主脉络：

1. Hook（0:00 - 0:20）
2. 问题引入（0:20 - 1:30）
3. 核心内容1（1:30 - 3:00）
4. 核心内容2（3:00 - 5:00）
5. 举例/案例（5:00 - 7:00）
6. 总结+升华（7:00 - 9:00）

#### 2. 分镜脚本
每一段都必须输出这些字段：

- `时间段`
- `节奏类型：高 / 中 / 低`
- `画面类型：A-roll / B-roll / 图示 / 动画`
- `内容描述`
- `剪辑动作`
- `字幕设计`
- `视觉锚点`
- `素材可得性：易 / 中 / 难`
- `素材方案：实拍 / 屏幕录制 / 免费素材 / AI图 / 图示 / 文字卡`
- `低成本替代方案`
- `信息密度：低 / 中 / 高`
- `注意力状态：拉高 / 稳定 / 消化 / 拉回`
- `目的`

### B. 剪映执行输出

剪映执行输出默认拆成三份，顺序不能反：

#### 文件 1：粗剪清单
目的：先让用户用最少操作，把片子剪到“顺、能看完、没有明显废话”。

粗剪清单只允许包含：
- 准备阶段
- 粗剪删废话
- 必要的少量重点放大
- 结尾是否保留稳定收束

粗剪阶段默认不做或少做：
- B-roll
- 大量关键词字幕
- 高频注意力变化

原则：
- 先保主线
- 先保口播可看性
- 不提前做太多装饰

#### 文件 2：精剪补充方案
目的：在粗剪完成后，再补足重点强化、B-roll、关键词字幕和注意力变化。

精剪补充方案只能做“加法”，不能推翻粗剪主结构。

允许补充：
- 重点句关键帧放大
- 必要 B-roll 画中画
- 关键词字幕
- 每 30 秒一次的轻量注意力变化
- 结尾慢推镜或节奏收束

禁止在精剪阶段要求用户：
- 大改主讲段落顺序
- 推翻粗剪已经删好的口播结构
- 大规模返工重切

#### 文件 3：最终检查清单
目的：在粗剪和精剪都完成后，用最短路径检查整条视频是否已经达到“能发”的状态。

最终检查清单不能新增复杂设计，只能做最后确认：
- 废话是否删干净
- 重点是否被看见
- B-roll 是否过多或挡字挡脸
- 关键词字幕是否太密
- 结尾是否足够稳
- 全片是否真的只用了允许的 5 种手法

最终检查阶段原则：
- 以检查为主
- 以小修正为主
- 不做大返工

每一份文件内部，都必须严格按这个顺序输出：

#### 一、准备阶段
- 打开剪映
- 新建项目
- 导入视频
- 自动生成字幕

#### 二、粗剪（删废话）
- 逐条说明哪些地方需要 `分割 → 删除`
- 明确判断标准：停顿、口头禅、重复句、卡壳、离题

#### 三、重点强化（放大）
- 从文案中提取重点句
- 标出具体句子
- 标出在哪段加关键帧放大
- 操作必须写成：`点击视频片段 → 播放头移动到句子开始 → 点“关键帧” → 缩放到 105%-120%`

#### 四、插入 B-roll（画中画）
- 只选必要位置，最多 6-10 个
- 每个位置必须写：
  - `时间/句子`
  - `搜索关键词`
  - `素材类型：实拍 / 截图 / 屏幕录制 / 免费素材 / AI图 / 文字卡`
  - `获取方式：自己拍 / 素材站搜索 / AI生成 / 用现有画面裁切 / 用文字卡替代`
  - `插入方式：画中画`
  - `放在什么位置`
  - `低成本替代方案`

#### 五、字幕优化
- 把长句改成 2-6 字关键词字幕
- 必须给出替换示例

#### 六、注意力控制
- 每 30 秒左右做一次变化
- 只允许两种方式：
  - 再次放大
  - 插入 1 秒 B-roll

#### 七、结尾处理
- 是否需要慢推镜
- 是否减少剪辑密度

## 剪映执行模式的硬性限制
整条视频只允许使用这 5 种操作：
- `分割 + 删除`
- `放大（关键帧 105% - 120%）`
- `插入B-roll（画中画）`
- `关键词字幕（2-6字）`
- `节奏变化（每30秒一次）`

禁止出现：
- 复杂特效
- 炫酷转场
- 多余动画
- 超出以上 5 项的花哨手法

## 工作流程

### Step 1：先读，不急着切
先把文案按“观点推进”拆段，而不是按字数平均切。

优先识别：
- 开场冲突
- 核心问题
- 两个主论点
- 抽象概念
- 比喻/案例
- 情绪变化点
- 结尾升华

### Step 1.5：先做内容骨架诊断
正式分镜前，先判断这条视频的内容结构是否足够清楚。

必须输出并使用这份诊断：
- 主问题是否明确
- 核心观点之间是否有推进关系，而不是并列堆砌
- 哪些段落需要案例，否则会变成空泛说理
- 哪些段落信息密度过高，需要加入消化段
- 哪些句子可以承担视觉锚点或关键词字幕
- 哪些画面设计必须降级为更容易获得的素材

如果文案结构不足，允许轻度重组段落顺序，但不能改掉用户的核心观点。

### Step 2：先设计节奏波形，再写镜头
先规划能量曲线，再安排镜头密度。

默认波形：
- `0:00 - 0:20` 高
- `0:20 - 1:30` 中
- `1:30 - 2:10` 高
- `2:10 - 2:40` 低
- `2:40 - 3:30` 中
- `3:30 - 4:10` 高
- `4:10 - 4:40` 低
- `4:40 - 6:20` 中
- `6:20 - 7:00` 拉高
- `7:00 - 8:00` 低
- `8:00 - 9:00` 中高收束

如果文案气质更克制，可以整体下调一个档位，但不能变成全程平。

### Step 3：按认知负担控制镜头密度
- 高密度段：2-4 秒一个变化
- 讲解段：4-8 秒一个变化
- 消化段：允许 8-12 秒稳定

不要机械执行“每 3 秒必须动一下”。
如果内容正在建立一个关键理解，宁可稳住画面，也不要为了热闹破坏理解。

### Step 4：抽象内容必须画面化
任何抽象概念，至少用下面 3 种方式之一落地：
- `画面隐喻`：把概念变成物体、空间、动作
- `生活例子`：把概念落到一个具体场景
- `图示对比`：用前后、左右、大小、路径表达差异

常见转译方法：
- “认知负担” → 屏幕同时弹出过多信息，人物停住，随后逐层清空
- “输出大于输入” → 水库出水量大于进水量，水位下降
- “知识没消化” → 胃里塞满食物却无法吸收

画面化时必须同时给出素材可得性判断：
- `易`：可用手机随手拍、屏幕录制、现有口播画面裁切、文字卡完成
- `中`：需要去素材站搜索，或需要简单 AI 图
- `难`：需要复杂实拍、复杂动画、难找的特定场景

默认优先选择 `易` 或 `中` 的方案。若某个画面是 `难`，必须提供一个 `低成本替代方案`。

### Step 5：每个关键观点都给视觉锚点
视觉锚点必须是观众能复述的东西，而不只是“有画面”。

优先使用：
- 一个反差对比图
- 一个记忆道具
- 一个明确场景
- 一个简单图示
- 一个可截图的金句版面

如果一个观点没有锚点，这段通常还没设计完整。

### Step 6：强制加入注意力回拉点
每 30-60 秒至少设计一次回拉点，方式可以是：
- 抛出新问题
- 突然切换到反例
- 字幕只留一句核心判断
- 镜头变静，语气突然放低
- 用“你可能以为...但真正的问题是...”做认知翻转
- 插入一个观众熟悉的具体场景

回拉点不是更吵，而是重新校准观众注意力。

### Step 7：保留消化段
连续输出抽象信息后，必须安排 8-12 秒的消化空间。

消化段常见处理：
- A-roll 稳定中近景，语速放慢
- B-roll 单一动作长镜头
- 图示只保留一个核心关系
- 字幕减少为 1-2 行

### Step 8：如果用户要剪映执行清单，先把分镜翻译成“点哪里”
不要输出抽象词，比如“加强节奏”“做一个视觉变化”。
必须改写成新手能照着点的动作。

写法要求：
- 用 `点击哪里 → 做什么`
- 一步只做一件事
- 尽量避免同一步包含多个判断
- 默认用户没有剪辑经验

### Step 9：把剪映执行拆成两轮协同
先输出 `粗剪清单`，再输出 `精剪补充方案`，最后输出 `最终检查清单`。

拆分原则：
- 粗剪解决“删什么、哪里收紧、哪里先轻放大”
- 精剪解决“哪里补 B-roll、哪里加关键词字幕、哪里做 30 秒注意力变化”
- 最终检查解决“发出去之前还要确认什么”
- 精剪不能和粗剪冲突
- 如果某项内容在粗剪阶段不是必须，就放到精剪阶段
- 如果某项内容不需要动刀，只需要确认，就放到最终检查清单

## 剪映执行模式输出规则

### 总体输出格式
剪映执行模式必须输出三份文件或三个大区块：

1. `粗剪清单`
2. `精剪补充方案`
3. `最终检查清单`

用户应该能先单独执行粗剪，再回来执行精剪，最后跑完检查，而不需要推倒重来。

### 一、准备阶段
固定写成操作说明：
- `打开剪映桌面版或手机版`
- `点击“开始创作”`
- `点击“导入”`
- `选中主讲人口播视频`
- `点击“添加”`
- `点击底部或右侧“文本”`
- `点击“识别字幕”或“智能字幕”`
- `选择“全部识别”`
- `点击“开始识别”`

### 二、粗剪规则
只删 4 类内容：
- 1 秒以上空白停顿
- 重复表达同一个意思
- 明显口头禅
- 讲错后重说的片段

每条都用这个格式：
- `把播放头移动到这句前面`
- `点击“分割”`
- `把播放头移动到废话结束位置`
- `再次点击“分割”`
- `点击中间这段废片`
- `按 Delete 或点击“删除”`

粗剪阶段补充要求：
- 全片优先只做 0-3 次轻放大
- 不强制先加 B-roll
- 先把节奏卡顺
- 结尾先保持稳定，不要过早堆变化

### 三、重点强化规则
只对重点句做关键帧放大。
全片建议 6-10 次，不要过多。

放大操作固定写法：
- `点击主轨道视频`
- `把播放头放到重点句开始`
- `点击“关键帧”`
- `把播放头向后移动 1-2 秒`
- `双指放大画面，或在缩放参数里调到 105%-120%`

如果句子结束要恢复：
- `把播放头移到句子结尾`
- `再次点击“关键帧”`
- `把缩放拉回 100%`

### 四、B-roll 规则
- 最多 6-10 个
- 每个 B-roll 默认 1-3 秒
- 插入方式一律 `画中画`
- 不要让 B-roll 盖住整段主讲人表达，优先短暂替换或局部覆盖

操作固定写法：
- `点击“画中画”`
- `点击“新增画中画”`
- `导入对应素材`
- `把素材拖到对应句子上方`
- `把时长裁到 1-3 秒`
- `把画中画放到画面下方或角落，避免挡住人脸和字幕`

精剪阶段补充要求：
- B-roll 默认都放在精剪阶段
- 如果粗剪里已经做过极少量 B-roll，精剪只补剩余位置
- 精剪新增内容不能破坏口播理解顺序

### 五、字幕规则
- 整句字幕保留给自动字幕轨
- 手动强化字幕只留 2-6 个字
- 每次只强调一个词组

### 六、注意力控制规则
每 30 秒左右检查一次，如果画面太久没变化，只做下面二选一：
- 给下一句重点加一次 105%-110% 的轻放大
- 插入 1 秒 B-roll

### 七、结尾规则
- 结尾优先减少切换
- 如果情绪要收束，可以做一次很轻的慢推镜
- 结尾最后 10-15 秒不要频繁加 B-roll

### 八、最终检查规则
最终检查清单必须像验收表。

固定检查项：
- `开头 15 秒能不能直接看下去`
- `有没有明显废话没删`
- `重点放大是否过多`
- `B-roll 有没有挡住脸或字幕`
- `关键词字幕是否都在 2-6 字`
- `每 30 秒左右是否有轻变化`
- `最后 15 秒是否足够稳定`
- `是否只用了 5 种允许手法`

如果发现问题，修正动作也必须写成：
- `点击哪里`
- `删什么`
- `移开什么`
- `把缩放改回多少`

## 输出规则

### 视频结构写法
先输出 `内容骨架诊断`，再输出 6 段视频结构，说明每段要完成什么认知任务、情绪任务、节奏任务。

内容骨架诊断必须短而具体，不要写成泛泛的内容评价。它的作用是指导后续分镜取舍。

### 分镜脚本写法
用清晰表格或分段卡片输出，保证剪辑师不需要二次翻译。

每段都要包含：
- 主画面是什么
- 为什么此时切换
- 字幕如何承载信息
- 观众此刻是被拉高、稳定、消化，还是被拉回
- 这个画面素材是否容易获得
- 如果素材不好找，最低成本替代方案是什么

### 剪映操作清单写法
像说明书，不讲理论。

必须满足：
- 每一步能在剪映里直接执行
- 多写“点击”“移动播放头”“分割”“删除”“画中画”“关键帧”
- 不写抽象空话
- 默认以最少操作做出“能看完”的成片
- 粗剪和精剪要明显分开，不能混在一起
- 最终检查清单要明显独立，像验收单

## 质量检查
输出前自检：

- 是否存在连续高强度剪辑超过 60 秒
- 是否每 30-60 秒有一次注意力回拉点
- 是否每个关键观点都有视觉锚点
- 是否所有抽象内容都已画面化
- 是否至少有 2 段明确的“消化段”
- 是否先输出了内容骨架诊断，并用它指导分镜取舍
- 是否标出了信息密度风险、案例/场景缺口、视觉化机会
- 是否每个 B-roll 都有素材类型、获取方式和低成本替代方案
- 是否优先使用低成本可获得素材，而不是默认要求复杂实拍或复杂动画
- 是否剪辑动作能被直接执行，而不是只有抽象描述
- 如果是剪映执行模式，是否每一步都能在 CapCut 里直接点出来
- 是否严格没有超出 5 种允许手法
- 是否已经分成“粗剪清单 + 精剪补充方案 + 最终检查清单”
- 精剪是否只做补充，而不是推翻粗剪
- 最终检查是否以确认和小修正为主，而不是重做

如果不满足，先修结构，再输出。

## 推荐风格偏好
- 讲解段以 A-roll 为轴，B-roll 只负责补偿，不抢主叙事
- 图示尽量简单，一次只解释一个关系
- 字幕不要把整段台词全贴满，优先突出判断句、关键词、数字、对比词
- 动画用于解释机制，不用于装饰
- 长视频更依赖“节奏起伏 + 镜头稳定感”，不是依赖高频快切

## 协作关系
- 上游常来自 `script-generator`
- 上游也可来自 `reference-script-rewriter`，用于把参考视频脚本重写成新脚本后再进入分镜
- 可与 `video-analysis` 协同，但本 Skill 优先处理长视频导演级分镜，不走短视频 14 镜头模板
- 下游可直接交给剪辑、配音、AI 出图或视觉规划环节

## 默认假设
- 用户要的是“可拍、可剪、可交付”的结果，不是理论分析
- 如果用户提到剪映、CapCut、不会剪辑、一步一步操作，优先使用 `capcut-execution` 或 `both`
- 如果用户明确提到“先粗剪，再补细节”，必须输出两份协同文件
- 如果用户提到“自动输出三份文件”“最后再检查一遍”，必须输出三份协同文件
- 若用户没有提供具体时长，默认输出 8-9 分钟版本
- 若原文案结构不足，允许先轻度重组再出分镜，但不要改掉核心观点

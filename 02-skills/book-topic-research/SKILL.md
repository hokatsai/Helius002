# Book Topic Research Skill

## 功能說明
圍繞一本書、一個書本概念或一個作者思想，整理成可供視頻創作使用的研究包。

這個 Skill 支持三種工作方式：

1. `公開資料模式`
   - 用戶沒有讀書筆記，也沒有本地書本文件。
   - 基於公開資料、作者訪談、研究摘要、書評和可信來源整理。
   - 必須標明「未直接分析全文」。

2. `讀書筆記模式`
   - 用戶提供自己的讀書筆記、摘錄、感悟。
   - 優先保留用戶自己的理解，再補充外部資料。

3. `本地書本模式`
   - 用戶提供本地 PDF / txt / md / epub 等文件路徑。
   - 先分析本地內容，再結合網上資料補全背景、反面觀點和視頻方向。

## 適用場景
- 想做一本書的視頻，但還沒有完整腳本。
- 想把書中概念轉成中文長視頻選題。
- 想基於自己的讀書筆記，加入研究資料和案例。
- 想先做《刻意練習》這類主題的素材包。
- 想找作者背景、核心理論、反對觀點、應用案例。

## 核心原則
- 如果沒有全文，不要假裝讀過整本書。
- 如果有用戶筆記，優先保留用戶自己的理解。
- 書本內容要轉成視頻主線，不是做章節摘要。
- 對書中概念要找現實案例和反面觀點。
- 不提供長篇受版權保護的原文摘錄。
- 引用只能短引，重點使用轉述和分析。
- 如果涉及研究，優先找作者原始論文、訪談、官方頁或可靠出版來源。

## 輸入
| 參數 | 類型 | 必填 | 說明 |
|------|------|------|------|
| book_title | string | 是 | 書名或核心概念，例如《刻意練習》 |
| user_notes | string | 否 | 用戶自己的讀書筆記、感悟、摘錄 |
| book_file_path | string | 否 | 本地書本文件路徑 |
| target_topic | string | 否 | 想聚焦的視頻角度 |
| output_dir | string | 否 | 默認 `04-outputs/{book-slug}-book-research/` |
| language_scope | string | 否 | `zh` / `en` / `both`，默認 `both` |
| generate_video_direction | bool | 否 | 默認 `true` |

## 輸出
默認生成獨立文件夾：

1. `01-book-notes.md`
   - 書本核心概念
   - 作者觀點
   - 如果有用戶筆記，整理用戶理解
   - 如果有本地書本，整理書本內容脈絡
   - 如果只有公開資料，明確標註資料來源限制

2. `02-research-notes.md`
   - 作者背景
   - 相關研究
   - 公開訪談/文章/書評
   - 案例素材
   - 反面觀點
   - 可視化比喻

3. `03-video-direction.md`
   - 暫定標題
   - 核心主張
   - 視頻結構
   - Hook 選項
   - 可用案例
   - 視覺方向

4. `04-source-list.md`
   - 來源鏈接
   - 每個來源用途
   - 可引用程度
   - 需要核實的地方

## 工作流程

### Step 1：判斷資料模式
先判斷本次使用哪種模式：

```text
如果有 book_file_path → 本地書本模式
如果沒有 book_file_path 但有 user_notes → 讀書筆記模式
如果兩者都沒有 → 公開資料模式
```

輸出中必須標明使用模式。

### Step 2：分析用戶材料
如果有用戶筆記：
- 提煉用戶原始理解
- 保留個人感悟
- 標出最適合做視頻主線的部分
- 標出需要外部資料支撐的地方

如果有本地書本：
- 讀取可訪問文本
- 提煉章節結構或核心概念
- 不長篇複製原文
- 對掃描 PDF 或無法讀取文件，提示需要 OCR 或轉文本

### Step 3：公開資料搜索
即使有本地材料，也要補充公開資料。

搜索方向：
1. 作者背景和訪談
2. 書的核心概念和爭議
3. 相關學術研究
4. 反面觀點或批評
5. 現實應用案例
6. 其他創作者如何表達類似主題

### Step 4：轉成視頻素材
不要只做讀書摘要。每個概念都要轉成視頻可用素材：

- 它能支撐哪一段？
- 可以用什麼生活例子？
- 有沒有反直覺 Hook？
- 有沒有可視化圖示？
- 觀眾可能會怎麼反駁？

### Step 5：形成視頻方向
默認輸出 1 條推薦主線。

視頻方向必須包含：
- 1 個推薦標題
- 3-5 個備選標題
- 1 句核心主張
- 6-8 分鐘視頻結構
- 2-3 個可用案例
- 反面觀點和平衡說法
- B-roll / 圖示方向

## 質量檢查
輸出前自檢：
- 是否標明資料模式？
- 如果沒有全文，是否避免假裝讀完整本書？
- 是否保留了用戶自己的理解？
- 是否把書本概念轉成視頻主線，而不是章節摘要？
- 是否補充了反面觀點？
- 是否列出來源和可引用程度？
- 是否避免長篇引用受版權保護原文？
- 是否保存到獨立 output 文件夾？

## 默認假設
- 用戶要的是視頻創作材料，不是純讀書報告。
- 沒有用戶筆記時，可以照常工作，但要標明基於公開資料。
- 有本地書本或讀書筆記時，優先分析本地/用戶材料。

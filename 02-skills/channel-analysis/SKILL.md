# Channel Analysis & Strategy Skill

## 功能说明
本 Skill 基于 Helius-002 架构，负责两大核心职责：
1. **分析层（Analysis）**：对 YouTube 频道进行深度拆解与竞品研究
2. **战略层（Strategy）**：建立账号顶层策略，明确定位、受众、边界与阶段目标

它融合了"8 大内容支柱"定位框架、爆款标题公式库、Topic Filter 选题逻辑以及账号战略规划，
旨在从内容生产力、流量获取力和商业变现力三个维度提升频道表现。

---

## 核心能力

### 分析能力（原有）
- 频道定位诊断（8 大内容支柱映射）
- 竞品拆解与差异化分析
- 爆款标题公式库生成
- Topic Filter 四重选题过滤
- 内容矩阵规划
- 变现策略建议

### 战略能力（新增 ⬆️）
- 账号定位文档生成（明确"我是谁"）
- 目标受众画像（明确"给谁看"）
- 内容边界定义（明确"什么不该做"）
- 阶段目标设定（涨粉 / 建信任 / 转化）
- 30 条方向清单生成（从支柱出发的可执行选题池）

## Strategy 模式详细说明（新增 ⬆️）

### 账号定位文档 `channel_positioning.md`
输出以下内容：
- 一句话定位（"我是一个 [做什么的] 频道，帮助 [谁] 解决 [什么问题]"）
- 核心价值主张（观众为什么选你而不是别人）
- 内容风格关键词（3-5 个形容词）
- 与竞品的差异化点（至少 3 点）
- 频道视觉/语言调性（正式/口语/幽默/深度等）

### 用户画像文档 `audience_persona.md`
输出以下内容：
- 目标人群画像（年龄、职业、收入、教育背景）
- 心理画像（焦虑点、渴望、痛点、价值观）
- 内容消费习惯（什么时间看、在哪看、看多长）
- 搜索行为（他们会搜什么关键词）
- 反面画像（什么人不是你的目标受众）

### 内容边界文档 `content_boundaries.md`
输出以下内容：
- ✅ 应该做的内容类型（含示例）
- ❌ 不应该做的内容类型（含理由）
- ⚠️ 灰色地带（需要谨慎评估的主题）
- 底线原则（绝不碰的话题/风格）
- 与内容支柱的对照检查表

### 阶段目标文档 `phase_goals.md`
输出以下内容：
- 当前阶段判断（冷启动 / 涨粉期 / 信任建设期 / 转化期）
- 当前阶段核心 KPI（订阅数 / 完播率 / 互动率 / 转化率）
- 当前阶段内容策略（侧重什么题型）
- 下一阶段目标与过渡条件
- 30/60/90 天里程碑

### 30 条方向清单 `topic_pool_30.md`
输出以下内容：
- 从每个内容支柱延伸 4-5 个具体选题方向
- 每个方向标注：题型（涨粉/转化/品牌）、难度、时效性
- 按优先级排序
- 标注哪些适合当前阶段

## 执行流程

### Analysis 模式
1. 获取频道数据（YouTube API 或手动输入）
2. 分析已有内容，归类到 8 大内容支柱
3. 识别定位偏差和机会缺口
4. 应用爆款标题公式生成候选标题
5. 使用 Topic Filter 四重过滤产出选题清单
6. 匹配变现路径
7. 输出报告到 `04-outputs/analysis/`

### Strategy 模式（新增 ⬆️）
1. 收集频道基础信息（定位、受众、竞品）
2. 执行 Analysis 获取现状诊断
3. 生成五份战略文档
4. 与现有 Topic Filter 结果交叉验证
5. 输出到 `04-outputs/strategy/`

### Full 模式
依次执行 Analysis → Strategy，合并输出

## 错误处理
### 错误类型与降级方案
| 错误类型 | 降级方案 | 重试机制 |
|----------|----------|----------|
| YouTube API 超时 | 使用缓存数据或手动输入代替 | 3 次重试 |
| 频道数据不足 | 返回基于现有数据的部分分析 | 不重试 |
| 网络断开 | 返回错误提示，保存草稿状态 | 3 次重试 |

### 错误日志
所有错误自动记录到 `06-logs/channel-analysis/YYYY-MM-DD.log`

# Content Review Loop Skill

## 功能说明

没有复盘的内容系统，即使发了 100 条视频也只是把错误重复了 100 遍。Content Review Loop（内容复盘闭环）模块，是 Helius-002 从“执行机器”跨越到“具有生命力的自我进化系统”的最终核心机制。

它接收你在真实市场（如 YouTube/B站/小红书等）发布内容后产生的数据反馈，通过多维度的归因分析（为什么火了，为什么扑街），将总结出的规律沉淀为经验库，从而直接修正和反哺前端的 `topic-scorer` (选题评分机制) 及 `channel-strategy` (频道战略定位)。

---

## 核心能力

- **多维数据回录**：记录发布后的点击率(CTR)、完播曲线（Retention Curve）、点赞率/转发数据。
- **正向爆款归因 (Success Pattern Analysis)**：如果视频数据超预期，拆解出是因为：踩中了当时的情绪密码？封面套路新颖？还是前 10 秒 Hook 留存极高？
- **逆向低谷归因 (Failure Pattern Analysis)**：如果视频数据不及预期，诊断是：标题点击率太惨淡（包装失败）？还是 30 秒内跳出率太高（开场拖沓）？或者是选题池严重脱落受众？
- **规则沉淀回流 (Feedback Loop Updater)**：强制输出更新后的`success_patterns.md` 或 `failure_patterns.md`。

## 斜杠指令

| 指令 | 功能 |
|------|------|
| `/review [数据文件/手动描述]` | 根据输入的后台数据，进行全面的复盘诊断，并输出下一步规避/复制建议 |
| `/analyze-drop [留存曲线数据]` | 专精于分析视频的完播率曲线，定位是哪里让人跑了（如：转场太慢、强说教、配图无聊） |
| `/analyze-ctr [曝光与点击数据]` | 专精于分析缩略图与标题组合的点击转化效能 |
| `/commit-rule [复盘结论]` | 将某一条黄金或者踩坑规则，永久写入 `channel-strategy` 或创作者风格指南中 |

## 分析诊断工作流 (Diagnostic Workflow)

### 诊断 1：点击率漏斗层 (CTR)
**数据表现：** 高曝光 + 极低点击率 (如 <3%)。
**系统归因推演：**
1. 是否标题缺乏“情绪唤醒（激动/恐惧/好奇）”？
2. 是否封面设计过于拥挤杂乱，核心大字超过了 6 个字？
3. 是否选题过于冷门，或系统错推给了非画像人群？
*联动：建议后续增强使用 `packaging-optimizer`。*

### 诊断 2：前 30 秒流失层 (Initial Retention)
**数据表现：** 点击率正常，但前 30 秒流失一半以上的观众。
**系统归因推演：**
1. 标题党嫌疑？视频开头与封面承诺产生断层。
2. 背景铺垫太长？没有使用最犀利的 Hook，打招呼和自我介绍废话太多。
*联动：建议检修 `script-generator` 的开场逻辑和起步语速。*

### 诊断 3：中途尿点/跳出层 (Mid-Video Drop-off)
**数据表现：** 数据在某一个时间点突然呈现断崖式下跌。
**系统归因推演：**
1. 是否在这里引入了大量枯燥的学术解释且缺乏白话比喻？（降维能力不足）
2. 是否长达一分半种缺乏 B-roll（空镜头）或画面转场切换，导致视觉疲劳？
*联动：反馈给 `video-analysis` 模块以增加视觉刺激点。*

## 数据格式与输出结构

### 输入示例 (用户通过 `/review` 提供的素材)：
```
视频：《不要再追求稳定了》
时长：8分30秒
数据情况：发布 24 小时播放量较平时激增 300%，CTR（点击率）高达 8.5%，但平均完播率只有 15%，大量用户在 3:15 处（解释“生成效应”底层逻辑时）流失。
```

### 系统的诊断输出：
```markdown
# 📈 表现复盘报告: 《不要再追求稳定了》

## 1. 核心表现界定
- **高光点**：点击率突破历史记录 (8.5%)，包装策略完全成功。
- **痛点**：有效留存崩塌，平均完播率堪忧。

## 2. 深度归因拆解
**成功归因 (Why it clicked)**：
使用了 `packaging-optimizer` 中的【恐惧/避坑】型策略。“稳定=毁灭”的反差概念极大地唤醒了职场人群的焦虑情绪，这是本期流量起爆的核心引擎。

**流失归因 (Why they left at 3:15)**：
经查该节点内容为“生成效应”与“费曼技巧”。普通观众对于没有马上给出解决方案，反而陷入长时间学术定理名词解释的内容极易失去耐心。在进入说理段落时，视觉与语言的降维没有接住。

## 3. 经验沉淀回流 (Next Action)
- [执行] 已将“反权威/痛点揭露（如：稳定=毁灭）”写入 `success_patterns.md`。
- [优化配置] 修改 `script-generator` 参数：未来在深度论述区（Evidence），严格禁止连续超过 100 字的学术概念直出，必须捆绑生活比喻；
- [视觉修正] 强烈建议下期视频在此类段落，必须配合使用高频快切空镜头（B-roll）来留住注意力。
```

## 与工作流协作（最终交接点）

随着 Content Review Loop 的每一次运行，你将会发现：
- 下一次使用 `topic-scorer` 评分时，会更倾向于给出这次跑通风格的高分。
- 下一次调用 `packaging-optimizer` 时，会自动排除掉这次证伪的错误封面排版。
此时，Helius-002 从单向的输出管线，正式形成了闭环。

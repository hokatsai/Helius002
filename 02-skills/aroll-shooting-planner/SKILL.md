# A-roll Shooting Planner Skill

## 功能說明
把視頻腳本、分鏡或視頻方向稿轉成可拍攝的 A-roll 口播計劃。重點是讓創作者在拍攝現場知道：分幾段拍、每段要說什麼、哪些句子需要重拍、鏡頭和收音要怎麼檢查。

這個 Skill 不負責寫完整腳本；它負責把已有內容變成拍攝執行文件。

## 適用場景
- 已有完整腳本，需要轉成提詞器和分段拍攝清單。
- 只有視頻方向稿，需要先做一版 A-roll 試拍計劃。
- 拍攝前需要燈光、構圖、收音、儀態檢查表。
- 想降低拍攝時的卡頓和重拍成本。

## 輸入
| 參數 | 類型 | 必填 | 說明 |
|------|------|------|------|
| script_or_direction | string | 是 | 完整腳本、視頻方向稿或分鏡 |
| duration_target | string | 否 | 默認 `6-10min` |
| shooting_style | string | 否 | 默認 `talking-head / 中文知識類口播` |
| teleprompter_mode | bool | 否 | 默認 `true` |
| output_dir | string | 否 | 默認寫入對應 `04-outputs/{中文主題包}/` |

## 輸出
默認輸出：

1. `04-aroll-shooting-plan.md`
   - 拍攝目標
   - 分段拍攝表
   - 提詞器段落
   - 表達節奏提示
   - 重拍標記
   - 現場檢查清單

## 工作流程

### Step 1：判斷輸入完整度
如果是完整腳本：
- 按語義和呼吸點切成 6-12 個 A-roll 段落。

如果只是方向稿：
- 先生成「試拍段落大綱」，不要假裝已有逐字稿。
- 每段給出要說的核心句和表達任務。

### Step 2：切分拍攝段落
每段控制在 30-90 秒，避免一鏡到底過長。

每段必須包含：
- `段落名`
- `預估時長`
- `表達任務`
- `開頭句`
- `必說句`
- `收尾句`
- `情緒/語速`
- `重拍提醒`

### Step 3：生成提詞器稿
提詞器稿要：
- 短句
- 多換行
- 保留口語停頓
- 不使用過長複句
- 對金句做標記

標記格式：
- `[重點]`：必須說準
- `[慢一點]`：需要降速
- `[停頓]`：給觀眾消化
- `[可重拍]`：這句容易卡，拍攝時單獨補一遍

### Step 4：拍攝檢查表
必須包含：
- 構圖
- 燈光
- 收音
- 鏡頭清潔
- 背景雜物
- 提詞器距離
- 眼神位置
- 錄製測試
- 備用電量/存儲空間

### Step 5：素材命名建議
給出簡單命名規則：

```text
A01_Hook_take01
A02_Problem_take01
A03_Point1_take01
```

## 質量檢查
- 是否能直接拿去拍？
- 是否避免把整篇稿一次拍完？
- 是否標出重點句和容易卡的句子？
- 是否有現場檢查表？
- 如果輸入不是完整腳本，是否清楚標明是試拍計劃？

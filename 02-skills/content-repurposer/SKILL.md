# Content Repurposer Skill

## 功能说明

Content Repurposer（多平台改写/一稿多发）模块，是基于创作者经济中的“内容瀑布流（Content Waterfall）”理论打造的效率杠杆引擎。

一篇精心制作的深度长内容（如 8 分钟 YouTube 长视频脚本）代表着极高的“内容资产”。如果只发布一次，是对资产的极大浪费。本模块能够提取母文本的核心逻辑，自动改写成针对不同平台用户消费习惯的短内容。

---

## 核心能力

- **长切短（Shorts / 抖音切片标记）**：不改变原意，提取最抓人的黄金 60 秒作为短视频独立脚本，包含 Hook 和极速反转。
- **长转图文（小红书 / 公众号笔记）**：将长视频的视听脚本语言，改写为高信息密度、带 emoji 和列表点状结构的图文语境。
- **长转短贴（Twitter / Threads帖）**：提取金句或某个具有冲突感的小论点，改写为 140 字以内极简输出。
- **长转通讯（Newsletter）**：调整语气，变成面向核心粉丝的私人对谈信件。
- **瀑布流联动**：深度结合 `dankoe` 的 P3 框架内容瀑布流（宏观->微观跨平台复用规则）。

## 斜杠指令

| 指令 | 功能 |
|------|------|
| `/repurpose [母脚本]` | 启动全矩阵改写，一次性输出四个平台维度的适配版本 |
| `/toshort [母脚本]` | 仅生成适合 Shorts / 抖音的高完播率竖屏短脚本 |
| `/toxhs [母脚本]` | 仅转换为小红书高赞图文笔记风格（带封面文字建议和标签） |
| `/totweet [母脚本]` | 从母稿中提取 1-3 条可用于 X/Threads 分发的短帖语录 |
| `/tonewsletter [母脚本]` | 改为一封深度沟通的 EDM / Newsletter 信件 |

## 改写规则引擎 (Repurposing Rules)

### 1. YouTube Shorts / 抖音 / 视频号 (竖屏短视频)
**逻辑转换**：长视频有时间做铺垫，短视频**绝对没有**。
- **结构阉割**：砍掉所有的背景介绍。第一句话必须是全片最有反差和冲击力的结论！
- **字数控制**：250字 以内（约 50-60 秒完播）。
- **语言**：比长视频语速快20%，每一句话都必须有信息量，或者是在推进悬念。必须口语化，严禁书面词汇。

### 2. 小红书 (图文/笔记)
**逻辑转换**：用户是在快速滑动，需要强烈的“点状信息输入”。
- **结构**：“痛点共情开场” + “1/2/3条干货点”（必须带黑体或序号） + “一句话金句总结”。
- **标点符号与 Emoji**：适量且恰当的 Emoji 作为视觉锚点。空行必须多，严禁大段不分行的阅读压迫感。
- **附带产出**：不仅给文案，还要列出【封面大字建议：xx】与【内页切片分布：共 3 张图】。

### 3. X (Twitter) / Threads (短信息流)
**逻辑转换**：激发互动，引发站队或认同。
- **结构**：一句话抛出带有一定挑衅感或违背大众常识的观点（Hook） -> 两句话解释底层逻辑 -> 留一个开放式问题或极端金句结尾。
- **字数控制**：总字数严格压缩，不废话，标点克制。

### 4. Newsletter (私人订阅信/公众号长文)
**逻辑转换**：面向已经信任你的死忠粉，需要展现私人思考过程。
- **人设调整**：降低强说教感，增加“我最近在反思…”“我这周遇到一件事…”的生活类开场。把视频里的“舞台感表达”换成“围炉夜话”式的对桌沟通。
- **结构保留**：完全保留母脚本深度，但展开更多私密感受。

## 典型使用链路

通常本模块作用于工作管线的倒数第二环节发布准备期：
1. 你的主线任务是完成并发布一个核心的 8 分钟宏观深度视频（依赖 `script-generator`）。
2. 调用 `/repurpose`。
3. 系统将为你拆解出明天发小红书的图文、后天发 Shorts 的短切片、大后天发推文的语录。
4. 一个长视频的投入，置换出一周的全矩阵流水线内容。

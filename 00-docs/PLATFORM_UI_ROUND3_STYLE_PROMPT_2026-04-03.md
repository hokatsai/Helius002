# Helius-002 平台 Demo 第三轮 UI 风格升级提示词

> 用途：给 Coding AI 直接读取，继续执行第三轮 UI 风格升级  
> 目标：在不改动产品结构与核心流程的前提下，把平台从“简洁原型感”升级为“办公型高级平台感”

---

## 一、这一轮的核心目标

前两轮已经完成：
- 平台骨架搭好
- Dashboard / Import / Analysis / Script / Library 已形成闭环
- demo flow 已经可跑通
- 第二轮正在往产品感推进

### 第三轮不要改产品结构
这一轮**不是**重做产品，
而是：

## 从现在这种
> 简洁、可演示、偏原型

升级成
> 更像成熟办公产品 / 创作者工作平台 / 现代 SaaS 后台

一句话：

**不改骨架，只改气质。**

---

## 二、这一轮绝对不要动的东西

### 不允许改动
- Dashboard / Import / Analysis / Script / Library 这 5 个模块名称与主结构
- Import → Analysis → Script → Library 这条 demo flow
- 核心交互逻辑
- Library 的资产沉淀定位

### 允许改动
- 页面布局比例
- 侧边栏风格
- 顶部区域风格
- 卡片系统
- 字体层级
- 间距与留白
- 颜色体系
- 阴影与边框
- 页面组织方式

---

## 三、风格参考方向

请将整个平台 UI 的设计风格升级为：

### 关键词
- 办公平台感
- 高级浅色系
- 现代 SaaS 后台
- 安静、克制、有秩序
- 不浮夸，但明显更成熟

### 参考感觉
- 更像工作台 / 控制台 / 任务平台
- 少一点“原型感”
- 多一点“真的会每天打开用”的可信度

### 参考风格说明
可以参考现代社群/课程/办公平台的结构气质：
- 左侧导航更稳
- 顶部栏更轻、更有秩序
- 内容区更宽阔
- 卡片更像产品组件
- 页面之间的视觉语言更统一

但注意：

**不要把它做成课程平台。**

你做的是：
**AI 创作者工作流平台**

所以语义仍然是：
- Dashboard
- Import
- Analysis
- Script
- Library

而不是 lesson / curriculum / course 这类语言。

---

## 四、视觉升级方向

## 1. 整体布局升级

### 左侧导航栏
当前问题：
- 可能还偏“轻原型感”

升级方向：
- 更稳重
- 更像真实办公平台侧边栏
- 具备更清晰的导航层级感
- 图标、间距、active 状态更专业
- 品牌区域更有识别度

### 顶部区域
升级方向：
- 更像专业工作台顶部
- 页面标题更强
- hint 更像辅助说明，而不是一段普通说明文字
- 任务状态 chip 更自然地融入右上角

### 主内容区
升级方向：
- 提升内容区留白
- 模块之间拉开层次
- 页面呼吸感更强
- 看起来更高级、更像办公产品

---

## 2. Dashboard 升级方向

### 当前问题
可能仍然偏 landing page / 欢迎页。

### 升级目标
把 Dashboard 做成：

**工作台首页 / 成果驾驶舱 / 创作者控制台**

### 具体要求

#### Hero 区
- 保留产品定位
- 但不再只是宣传区
- 更像任务入口 + 平台总览

#### 增强“驾驶舱感”
请强化以下组件的视觉表现：
- 最近任务
- 工作流状态
- 已保存资产概览
- 最近产出预览

### 视觉目标
打开首页时，观众第一感觉应该是：

> 这是一个内容工作平台，不是一个演示 landing page。

---

## 3. Import 页升级方向

### 当前问题
功能已经有，但视觉上可能仍然更像输入表单。

### 升级目标
把 Import 页做成：

**任务创建器 / 工作流启动页**

### 具体要求
- URL 导入区更像一个正式模块
- 文本导入区更像一个正式模块
- 两者之间视觉区隔更明确
- 右侧状态栏更像“任务面板”
- Step 进度感更强

### 感受目标
不是让用户觉得在“填写一个表单”，
而是：

> 正在启动一个 AI 工作流任务。

---

## 4. Analysis 页升级方向

### 当前问题
信息有了，但视觉可能还是偏“区块堆叠”。

### 升级目标
把 Analysis 做成：

**内容拆解工作台 / 分析报告面板**

### 具体要求
- Summary 与 Why it works 的区块更高级、更有主次
- Key Insights 更像结论输出，不像普通列表
- Viral Structure 更像步骤卡/结构卡，而不是简单文字块
- CTA（生成脚本）更有推进感

### 感受目标
让观众一看就明白：

> 这个平台真的在帮我拆内容，而不是只是显示一些文字。

---

## 5. Script 页升级方向

### 当前问题
逻辑完整，但结果页的“交付感”还可以更强。

### 升级目标
把 Script 页做成：

**结果交付页 / 可直接使用的产出页**

### 具体要求
- 标题候选更像“可选结果卡”
- 脚本正文更像可阅读、可复制、可交付的编辑页面
- 结构标签（Hook / Evidence / Core / Twist / CTA）更醒目
- 复制脚本 / 保存到素材库的按钮更像最终动作
- 可以加入小型状态标签，如：
  - Ready to use
  - Script generated
  - Draft saved

### 感受目标
观众看到这一页时应觉得：

> 这不是草稿区，而是已经能交付给创作者使用的结果页面。

---

## 6. Library 页升级方向

### 当前问题
现在更像任务记录页，还不够像资产库。

### 升级目标
把 Library 做成：

**内容资产沉淀区 / 内容仓库 / 创作者资产库**

### 具体要求
- 卡片更像资产条目，而不是普通列表
- 信息层次更明确
- 各种 snapshot 更有“产品化表达”
- 强调：来源、时间、结果、状态

### 每条资产建议更清晰展示
- 任务名称
- 来源类型
- 时间
- Analysis snapshot
- Script snapshot
- 状态标签（Saved / Ready）

### 感受目标
观众看到这一页会觉得：

> 这个平台会把一次次任务沉淀成长期可复用的内容资产。

---

## 五、颜色 / 组件 / 细节要求

### 颜色方向
- 浅色背景为主
- 更有层次的灰度系统
- 可加轻微品牌色点缀，但不要花哨
- 强调“办公感”，不是“设计作品集感”

### 卡片系统
- 卡片更统一
- 更有秩序感
- 阴影更克制
- 圆角适中
- 信息密度更专业

### 按钮系统
- 主按钮更有权重
- 次按钮更克制
- hover 更自然
- 不要过度动效

### 字体层级
- 标题 / 副标题 / 辅助文字层次更清楚
- 页面标题要更有重量
- section title 更有模块感

### 留白与间距
- 提升整体呼吸感
- 不要显得“太素”或者“太空”
- 要做到：简洁但不单薄

---

## 六、这一轮不要犯的错误

### 不要这样做
- 不要改成课程平台语义
- 不要做得像教育产品
- 不要大幅改交互路径
- 不要再加太多新功能
- 不要搞花哨炫技视觉
- 不要重渐变、重阴影、重装饰

### 这一轮真正要做的是
- 让现有结构更高级
- 让现有页面更成熟
- 让观众更信服它是一个产品

---

## 七、第三轮成功标准

完成后，这个平台应该达到：

### 第一眼观感
- 像一个成熟的办公型创作者平台
- 不再有强烈原型感

### 页面感受
- Dashboard 像控制台首页
- Import 像任务启动器
- Analysis 像拆解工作台
- Script 像结果交付页
- Library 像资产沉淀区

### 舞台效果
观众打开后会觉得：

**“这已经不是一个简单 demo，而是一个有产品形态的创作者工作平台。”**

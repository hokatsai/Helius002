# Packaging Optimizer Skill

## 功能说明

Packaging Optimizer 是 Helius-002 的"点击率提升引擎"。哪怕内容极其优秀，如果没有好的包装，也不会有人点开。本模块专注于解决**"让人愿意点开看前 3 秒"**的核心问题。

它通常在 `script-generator` 完成脚本后介入，进行打包润色与跨平台适配验证。

---

## 核心能力

- **多变体标题生成（Title Variations）**：一键生成 10+ 互斥风格的标题
- **缩略图视觉文案（Thumbnail Text）**：生成字数极少、冲突极强的大字文案
- **前 3 秒钩子优化（Opening Hooks）**：根据最终标题，反向微调或提供 3+ 个视频开场白
- **跨平台适配（Platform Adaptation）**：针对 B站/YouTube/Shorts/小红书 输出不同话语体系的标题
- **一致性校验（Consistency Check）**：校验"标题党"指数，确保观众点进来能通过前 30 秒留住
- **风格库联动**：深度调用 `triangle-narrative` 及 `dankoe` 等配置库内的经过验证的标题公式

---

## 斜杠指令

| 指令 | 功能 |
|------|------|
| `/package [脚本/主题]` | 全量进行一波打包：输出 10 个标题 + 5 个封面文案 + 3 个 Hook |
| `/title [主题]` | 仅生成多风格标题库（10+） |
| `/thumbnail [主题/标题]` | 仅基于给定前提生成封面大字（3-5字极简规则） |
| `/adapt [现有标题]` | 将一个标题改写适配目前主流的 4 大平台 |

## 打包要素详解

### 要素 1：高点击标题（Titles）

系统不仅输出标题，更会对标题的**核心动机**进行切分，包含但不限于：
1. **反直觉/颠覆型**（打破常规认知）
2. **利益驱动型**（指出立刻能获得的价值）
3. **恐惧/避坑型**（错过或做错会引发的后果）
4. **好奇/缝隙型**（利用信息差留下悬念）
5. **数字/盘点型**（强化结构感与掌握感）

**约束规则**：
- 严格引用 `triangle-narrative` 或其他已加载创作者的标题公式公式。
- 字数控制：YouTube 端最好<45 字符，B站端最好<25 字。

---

### 要素 2：缩略图/封面大字（Thumbnail Text）

**视觉法则**：封面字**绝对不是标题的重复**！如果标题是抛出问题，封面就是极端答案；如果标题是陈述，封面就是反问情绪。

**约束规则**：
- **极简**：严格控制在 **3-6 个字** 以内（手机端缩略图可读性）。
- **放大情绪**：能用"别傻了"就不用"请注意"。
- **视觉联想**：搭配生成的文字，提供应该配什么画面的建议（配合 `video-analysis` 的 Visual Plan 使用）。

*示例*：
> 标题：为什么最聪明的年轻人都在逃离稳定
> 封面大字：**稳定 = 慢性自杀** + [配图：一个背影走入迷雾]

---

### 要素 3：前 3 秒口播钩子（Hooks）

好标题把人骗进来，这 3 秒钟要把人锁住。Hook 必须与标题直接承接，不能有任何废话打招呼。

**校验逻辑**：
- 如果标题是抛出问题，Hook 的第一句必须直接开始解答或给出最反常的结论。
- 如果标题是展示现象，Hook 第一句必须抛出极端的诱因。

---

### 要素 4：多平台适配机制

相同的内核体系，不同平台的标题受众偏好完全不同：

| 平台 | 偏好的包装语境 | 示例转换 |
|------|--------------|----------|
| **YouTube (长视频)** | 好奇悬念、高质量纪录片感、数字盘点、强反差 | "纳瓦尔的 3 个致富秘密（大多数人不知道）" |
| **B站 (横屏解说)** | 玩梗、带点吐槽感、深度拉满、学术降维、情绪价值 | "被骗了 20 年？资本家最怕你搞懂的【残酷真相】" |
| **小红书 (图文/竖屏)** | 极度利己、干货密集、姐妹/兄弟快看、标签化 | "搞钱必看🔥普通人逆袭的 3 个底层逻辑！" |
| **Shorts/抖音** | 一秒入戏、极限反转、痛点直戳 | "停下！如果你还在追求稳定，你正在毁了自己..." |

## 与工作流协作

1. `topic-scorer` 产出好选题
2. `script-generator` 产出好剧本
3. **`packaging-optimizer`** 给剧本套上不可抗拒的包装
4. (`video-analysis` 接手开始按包装规划视觉)

### 被依赖与依赖
- **依赖源**：`script-generator` 的输出结果，`triangle-narrative` 内部定义的爆款标题库
- **后续动作**：可以直接交由系统自动落盘为 `04-outputs/packaging/{topic_name}.md`

## 错误处理
- 脚本信息量过小：如果传入内容少于 50 字，会提示要求提供更多上下文，防止生成过度悬浮的"标准正确废话"标题。
- 标题党惩罚：内置自我检验 prompt。如果检查出封面/标题承诺的利益点在提供的 script 中没有兑现，会主动降级或重写。
